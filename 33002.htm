33002: Floating-Point exception

Info: EPC
Data type: UDINT

Description:
Runtime error while processing R4 data if the floating point unit (FPU) detects an exception condition.
Such exceptions arise from calculations with invalid floating point numbers or when the result exceeds the valid range. This error is registered only once and only on the 1. ACOPOSmulti axis (channel).
Valid range of 32bit floating point numbers: See error number 73.

Reaction:
The faulty calculation is aborted with an invalid result. The reaction depends on the further processing of this invalid data.

Cause/Remedy:
Invalid floating point numbers while writing parameters, sending cyclic data to the drive or while receiving data from a network coupling.
Invalid numbers can result e.g. in following cases:
- Calculation with 64bit floating point numbers (double float) and conversion (cast) to float.
- Using an integer number within the data area of a float number (wrong mapping).
Large position jumps on inputs of function blocks or on the axis of the cam profile automat - e.g. caused by a homing procedure when a master/slave coupling is still active.
Extreme design of a cam profile in conjunction with a very high master velocity.
