5038: Homing procedure mode not possible: Restore data invalid

Info: Detail
Data type: UINT

Description:
Response error using 'start homing procedure' command. The configured homing mode (ncRESTORE_POS) restores the unit position from a permanent memory area. This memory area is empty or consists of invalid data.

Reaction:
The homing procedure is not started.
The 'homing position valid' state is reset.

Cause/Remedy:
No data in permanent memory area:
- New ACOPOS-HW
- Activation HOMING_RESTORE_MODE=1 is missing
Data of permanent memory area does not match the encoder/unit configuration:
- Encoder was changed
- Configuration is incorrect or was changed (ENCOD_COUNT_DIR, SCALE_LOAD_UNITS, SCALE_LOAD_MOTREV, SCALE_ENCOD_INCR)
Data of permanent memory area destroyed (Checksum)
-> Calibrate axis with "normal" homing procedure, this means a homing mode other than ncRESTORE_POS
