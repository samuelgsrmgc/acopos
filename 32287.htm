32287: Extended coupling data not possible (OutTime difference too large)

Info: Difference between maxOutTime and minOutTime
Data type: UDINT

Description:
Extended coupling data are used then for a network interface, if "Extended coupling data for ACP10_MC_BROADCAST=Yes" is defined for this network interface in the NC configuration.

Extended coupling data can only be used if it is guaranteed that the coupling data can be received by the ACOPOS in each coupling cycle in synchronization with the network. Among other things (see also error number 32288), this requires that the difference between the maximum transmission time (maxOutTime) and the minimum transmission time (minOutTime) is lower than the cycle time of the coupling cycle.

The cycle time of the coupling cycle is the maximum of the following two cycle times:
- Cycle time of the NC manager task class
- POWERLINK cycle time of the corresponding network interface

If the difference between maxOutTime and minOutTime is greater than or equal to the cycle time of the coupling cycle, then this error is indicated.

Reaction:
Initialization of extended coupling data is aborted.

Cause/Remedy:
The difference between maxOutTime and minOutTime depends on the system configuration of the NC manager task class and the POWERLINK configuration of the corresponding network interface.

System configuration of the NC manager task class:
The difference can be reduced by selecting one of the following settings for the parameter "I/O output delay":
- delay to middle of cycle
- delay to end to of cycle, only for task class Cyclic#1

POWERLINK interface configuration, I/O bus parameters:
The difference can be reduced by selecting the NC manager task class for the parameter "Bus cycle trigger".
