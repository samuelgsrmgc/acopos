32208: Error using plAction(DEVICE_TO_BUS_NR) (additional info in Logger)

Info: Status of plAction()
Data type: UINT

Description:
For the operation of ACOPOS servo drives via the POWERLINK network, among other things the function plAction(GET_PDO_INFO) is called. This error number is output if an error occurs with this procedure.

Reaction:
The ACOPOS startup function is aborted for this ACOPOS.

Cause/Remedy:
Correct the POWERLINK error listed in the Info.

Starting with Automation Studio V4.0, it is possible to switch to CPU simulation based on ARsim when using a real CPU as a target platform without the settings for ACOPOS simulation having to be changed.

If CPU simulation based on ARsim is used with ACP10 versions before V2.380, then this error number with Info 20903 is displayed for all ACOPOS axes for which the following is valid:
- ACOPOS Simulation is switched on for this axis
- ACOPOS Simulation is switched off for other axes on the same POWERLINK interface
To avoid this problem, ACOPOS simulation must be switched on explicitly for all axes in the NC mapping table.

This problem no longer occurs when switching to ACP10 version V2.380 or higher since ACOPOS simulation is automatically switched on when using ARsim with these ACP10 versions.
