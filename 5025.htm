5025: Homing offset with counting range correction cannot be set

Info:
Data type:

Description:
Response error using 'start homing procedure' command. A homing mode was defined with counting range correction (+ncCORRECTION). Automatic correction of the homing offset cannot be performed because the encoder counting range with overflow is not unique in the entire movement range.

Reaction:
See error number 5017.

Cause/Remedy:
The maximum encoder counting range is smaller than the movement range between the SW limits.
-> Difference between the SW limits is too large. Load scaling (units per motor revolutions) is too small. No absolute encoder (incremental encoder), encoder counting range is too small (e.g. only 'Single-Turn').
The testing limits exceeded the Integer32 counting range.
-> Value for the SW limits or load scaling is defined too large.
