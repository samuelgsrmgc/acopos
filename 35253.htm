35253: SMC: Encoder - Rounding error due to parametrization

Info: Additional info
Data type: UDINT

Description:
Error during startup of the encoder. The length configured in "Length of physical reference system for linear encoder (nm)" can not be represented as an integer multiple of encoder increments.

In the additional information (Info1) the length of an increment is displayed.

The unit of resolution of the encoder is shown in the Info0. This information is only available in the safety logger.
0 ... nanometer
1 ... picometer

Offset information for safety logger:
binary data[16..15]: Info0
binary data[20..17]: Info1

Reaction:
The status SafePositionValid is set to FALSE.
The status NotErrENC is set to FALSE.
If a safety function is used which needs an encoder, then the module changes into Functional Fail Safe state.

Cause/Remedy:
Check the parameter "Length of physical reference system for linear encoder (nm)" in the SafeDesigner or use another encoder!
