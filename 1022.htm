1022: Timeout in life sign monitoring of cyclic data to drive

Info:
Data type:

Description:
Runtime error during active life sign monitoring and active controller, if no cyclic data to drive arrives for a specified amount of time (CYCLIC_TODRV_T_CTRL).

Reaction:
Default setting STOP_ERR_DECEL_RAMP=ncA_LIMIT:
When position control is active (CONTROLLER_MODE=1 or 3):
Movement stopped with the axis limits and by switching off the controller.
For different controller modes see also error number 9000.

For different settings of STOP_ERR_DECEL_RAMP see error number 4007.

Cause/Remedy:
Problem in life sign monitor:
Time for life sign monitoring CYCLIC_TODRV_T_CTRL is defined too short.
Life sign monitoring and controller are activated too early, before cyclic telegrams are sent.

Problem in data generation and data transfer:
The counter within the cyclic user data to drive is not increased (CYCLIC_TODRV_COUNT).
Cycle time exceeded, extremely high computing time load or failure of the controller task.
Disturbance or disconnection of the network connection. See wiring guidelines error number 7030.
The network load is too high.
