6030: Holding brake: Brake output is active, but no brake entered in motor data

Info:
Data type:

Description:
Runtime error using the 'switch on controller' command when holding brake monitor is active. Current is flowing to the holding brake output even though a holding brake was not specified in the motor parameter block.

Reaction:
See error number 6045.

Cause/Remedy:
Incorrect configuration of the holding brake in the motor parameter block.
The output for the holding brake is being used for another functionality and the brake monitor has not been deactivated (bit 2 in the BRAKE_MODE not set).
