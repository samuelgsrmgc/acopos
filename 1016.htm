1016: Maximum cycle time exceeded - CPU load too high

Info:
Data type:

Description:
Runtime error while processing the mid-priority operating system functions if the computing time load on the ACOPOS is too high (e.g. set value generation, network processing and function blocks are no longer completed with the position controller cycle of 400us). The high priority controller functions are further processed with a rigid cycle.
ACOPOSmulti, ACOPOSmicro: This error is only registered on the 1. axis (channel).

Reaction:
Depending on the duration and the degree by which the cycle time is exceeded, subsequent errors can occur (e.g. speed jumps and lag errors).

Cause/Remedy:
For diagnosis see parameter CPU_TICKS_CYCLE_PEAK (139).
The following items show some common examples, which increase the processor load:
Increased switch frequency F_SWITCH.
Increased current controller dynamics (halving current controller cycle time) ICTRL_MODE Bit0.
Set current filters are active ISQ_FILTER1_TYPE, ISQ_FILTER2_TYPE, ISQ_FILTER3_TYPE.
Many function blocks active FUNCTION_BLOCK_CREATE. Simultaneous events at several EVWR FB instances or writing a computation intensive command produces load peaks.
Cyclic communication, data to drive CYCLIC_TODRV_PARID. Writing several paramter IDs simultaneously or writing a computation intensive command produces load peaks.
Cam automat, compensation gears AUT_COMP_MODE.
Virtual axis, basis movements or cam automat.
Several network coupling functions are active CONFIG_MA1/2/3/4/5_CYCLIC_POS.
Virtual encoder active ENCOD0_MODE.
Repetitive Control active CMD_SCTRL_RC.
Cyclic Trace, recording several test data points.
ACOPOS: Additional plug-in card in slot3 or slot4. Encoder plug-in cards need a little more computation time.
ACOPOSmulti: Both axis (channels) are active.
