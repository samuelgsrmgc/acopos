5102: Too many changes of cam profile per cycle (master period too short)

Info:
Data type:

Description:
Runtime error when cam automats are active if consecutive scan cycles (calculation cycle 400us) require a cam profile change each time. A maximum of one cam profile change is executed in a scan cycle. If this maximum 'frequency of change' is no longer sufficient, then the cam profiles lose their shape and the position relationship between the master and slave is lost.

Reaction:
See error number 4008.

Cause/Remedy:
Speed too high in the master axis or an additive axis.
Position jumps in the master axis or an additive axis.
Master period of a cam profile or a compensation gear too short.
Master multiplication factor too small.
Change-of-state event with the attribute ncAT_ONCE occurs too often.
See also error number 5036.
