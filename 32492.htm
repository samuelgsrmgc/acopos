32492: ACOPOS Simulation: TC for cyclic PLCopen data not equal to NC Manager TC

Info:
Data type:

Description:
For SG4 targets that task class can be defined, in which the cyclic PLCopen data are to be copied from a program to the POWERLINK network and vice versa.

Global this is possible in the ACP10 configuration (acp10cfg) for each POWERLINK interface with the parameter "Task class for handling of cyclic data with PLCopen (only for SG4)". The value contained in acp10cfg is entered in the network trace for each POWERLINK interface with node number 0 with ParID TASKCL_CycDat_PLCopen.

Local for each ACOPOS communication channel this task class can be defined in the NC mapping table in the data record of a real or virtual axlis in the column "Additional Data" with the XML attribute PLCopen_CyclicData_TaskClass. The values defined with PLCopen_CyclicData_TaskClass are entered in the network trace for the corresponding NC objects with ParID TASKCL_CycDat_PLCopen.

If ACOPOS simulation is activated, then the task class for handling of cyclic data with PLCopen must be equal to the task class for NC manager task (TASKCL_CycDat_PLCopen zero or same number as the task class for NC manager task). If another value is defined, then this error is indicated.

Reaction:
The ACOPOS startup function is aborted for this ACOPOS.

Cause/Remedy:
Correct the value in the ACP10 configuration or in the NC mapping table.
