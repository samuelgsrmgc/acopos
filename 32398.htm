32398: acp10sys does not contain an operating system for this ACOPOS hardware type

Info:
Data type:

Description:
The BR module "acp10sys" should contain a device-specific operating system for every type of ACOPOS hardware unit that can be connected to the network. This enables the device-specific operating system to be transferred to the ACOPOS module during ACOPOS startup if it is not already present on the ACOPOS unit.

If during ACOPOS startup the NC manager detects that the BR module "acp10sys" does not contain the device-specific operating system for an ACOPOS hardware type currently connected to the network, then this error will be displayed in the following situations:
- No operating system present on the ACOPOS
- The operating system version on the ACOPOS is not compatible with the NC manager version

Reaction:
The ACOPOS startup function is aborted for this ACOPOS.

Cause/Remedy:
"acp10sys" does not contain an operating system for the device type of the connected ACOPOS hardware module.
A version of the ACP10 software that contains the device-specific operating system for this ACOPOS hardware type must be added to the project.
