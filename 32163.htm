32163: A system module download to all drives is not possible with SwNodeSelect

Info:
Data type:

Description:
A system module download to all ACOPOS nodes connected to the network was started. If an ACOPOS node is active on the network, for which "SwNodeSelect" (node number selection via software) was defined, then a system module cannot be transferred to this ACOPOS for technical reasons (node number "lost" after SW Reset).

Reaction:
Transfer of the system module to the ACOPOS is aborted.

Cause/Remedy:
An ACOPOS node on the network is active, for which "SwNodeSelect" was defined, while "acp10sys" is being transferred from Automation Studio to the PLC as NC system module.

This problem can only occur for SG3 target systems and only with the following setting:
- Store NC operating system to the PLC: No (default setting for SG3)

The following setting is absolutely necessary for "SwNodeSelect":
- Store NC operating system to the PLC: Yes
