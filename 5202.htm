5202: Cam Control: Switch positions not in ascending order

Info:
Data type:

Description:
Response error using the drum sequencer (CMD_DRUMSEQ=ncSTART) or using the cam controller function block (CAMCON_MODE, CAMCON_CAM_S2). The start position of a cam in the interval is larger than the end position or the positions of neighboring cams overlap.

Reaction:
The drum sequencer (CMD_DRUMSEQ) is not started.
The parameter value (CAMCON_CAM_S2) is not accepted.

Cause/Remedy:
Cams were not deactivated when reconfiguring or the number of cams was not reduced ('old cams' remain).
The configuration order was switched (first the start position, then the end position).
